# Генератор КП — ESCORE

Статичний сайт (HTML/CSS/JS, без збірки й сервера — той самий підхід, що й у
`ses-calculator`), який формує брендовану комерційну пропозицію на СЕС з:

1. посилання на Google Sheet з розрахунком (вкладки **ПДВ** і
   **Моделювання Фін. показників роботи СЕС**);
2. PDF-звіту з розрахунком генерації (опційно — вставляється як додаток);
3. зображень розташування панелей / візуалізації об'єкта.

На виході — сторінка з готовим КП, яку друкують у браузері (Ctrl/Cmd+P →
«Зберегти як PDF»), точно як зараз влаштована вкладка «КП» в
`ses-calculator`.

## Швидкий старт (локально)

Файл `index.html` можна відкрити прямо в браузері (`file://`), без сервера.
Для повноцінної роботи знадобиться один раз вписати ключ Google API — див.
нижче.

## Розгортання на GitHub Pages

1. Створи новий репозиторій на GitHub (наприклад
   `EscoreOdessa/kp-generator`) і залий туди вміст цієї теки.
2. У налаштуваннях репозиторію **Settings → Pages** вибери джерело — гілка
   `main`, коренева тека.
3. За кілька хвилин сайт буде доступний за адресою
   `https://escoreodessa.github.io/kp-generator/`.

Хостинг безкоштовний, оновлення — просто новий `git push`.

## Налаштування Google Sheets API (один раз)

Щоб сайт міг читати дані прямо з Google Sheets, потрібен ключ Google Cloud
API з увімкненим **Google Sheets API**. Можна перевикористати той самий
проєкт Google Cloud, що вже створений для Maps API в `ses-calculator`.

1. Зайди в [Google Cloud Console](https://console.cloud.google.com/) →
   вибери той самий проєкт, що й для Maps API (або створи новий).
2. **APIs & Services → Library** → знайди «Google Sheets API» → **Enable**.
3. **APIs & Services → Credentials** → або створи новий API-ключ, або
   відкрий існуючий (той, що для Maps) і додай йому дозвіл на Sheets API.
4. Відкрий `js/config.js` і встав ключ у `GOOGLE_API_KEY`.

Важливо: файл-розрахунок у Google Sheets має бути відкритий «для всіх, у
кого є посилання» (View) — так само, як зараз відкритий приклад Grand
Marine. Без цього API-ключ не зможе прочитати таблицю (сайт не використовує
твій особистий логін в Google, тільки публічний доступ за посиланням).

## Якщо назви вкладок або колонок зміняться

Парсер у `js/sheets.js` навмисно шукає потрібні дані **за текстом
заголовків**, а не за жорсткими адресами комірок (A1, C14 тощо) — це
зроблено так, щоб дрібні зміни у файлі-розрахунку (нова колонка, інший
порядок) не ламали генератор. Якщо все ж КП перестане підтягувати якісь
цифри:

1. Перевір назви вкладок у `js/config.js` (`SHEET_TAB_PDV`,
   `SHEET_TAB_MODEL`) — вони мають точно співпадати з назвами вкладок у
   Google Sheets.
2. Подивись ключові слова пошуку в `js/sheets.js` (коментарі там же) —
   можливо, треба додати ще один варіант формулювання заголовка.

## Що ще варто знати

- **PDF з розрахунком генерації** зараз використовується як
  додаток-підтвердження (перша сторінка вставляється як картинка в розділ
  «Технічне рішення») — самі цифри для розрахунків (генерація, економія,
  окупність) беруться з вкладки «Моделювання», бо там вони вже пораховані
  й надійні. Якщо потрібно автоматично витягувати конкретні цифри саме з
  цього PDF — знадобиться зразок реального файлу, щоб написати парсер під
  його структуру.
- **Зображення** (розкладка панелей, візуалізація) — перше завантажене
  зображення йде на обкладинку, решта — у галерею на сторінці «Про
  проєкт».
- Кнопка **«Використати демо-дані»** на формі дозволяє перевірити вигляд
  документа без підключення до Google Sheets (корисно для навчання нових
  співробітників чи діагностики).
- Прив'язки до реквізитів компанії (адреса, телефон, умови оплати за
  замовчуванням тощо) — у `js/config.js`.

## Структура проєкту

| Файл | Призначення |
|---|---|
| `index.html` | Форма завантаження + контейнер документа |
| `style.css` | Брендовані стилі + верстка під друк (A4, розриви сторінок) |
| `js/config.js` | API-ключ, назви вкладок, реквізити компанії, бізнес-правила за замовчуванням |
| `js/logo.js` | Логотип ESCORE (base64 PNG) |
| `js/sheets.js` | Читання Google Sheets (вкладки ПДВ / Моделювання) через Sheets API |
| `js/pdf-report.js` | Обробка завантаженого PDF-звіту генерації (pdf.js) |
| `js/images.js` | Читання завантажених зображень |
| `js/kp-render.js` | Побудова 7 сторінок КП + діаграма (Chart.js) |
| `js/app.js` | Зшиває форму, парсинг і рендер |
