# Генератор КП — ESCORE

**Живий сайт:** https://escoreodessa.github.io/kp-generator/

Статичний сайт (HTML/CSS/JS, без збірки й без сервера — той самий підхід, що й у `ses-calculator`), який із посилання на файл-розрахунок у Google Sheets формує брендовану комерційну пропозицію (КП) на сонячну електростанцію (СЕС).

Менеджер вставляє посилання на розрахунок, за потреби додає зображення та кілька додаткових посилань, обирає формат і режим — і отримує готове КП, яке зберігає у PDF прямо з браузера.

---

## Що потрібно на вході

Обов'язкове — лише **посилання на Google Sheet з розрахунком**. Усе інше опційне:

- **Посилання на файл-розрахунок** (Google Sheets) — головне джерело цифр. Використовуються вкладки `ПДВ` (або друга номенклатурна вкладка для режиму «C», див. нижче), `Моделювання Фін. показників роботи СЕС` (фінансові показники), а для «Розширеного бюджету» — ще й вкладка «Кошторис».
- **Зображення розташування панелей / візуалізація** — одне або кілька (PNG, JPG тощо). Перше йде на обкладинку, решта — у галерею.
- **Посилання на звіт PVsyst.pdf** (Google Drive, опційно) — додає сторінку «04» зі скріншотом сторінки звіту (карта затінення / 3D-модель об'єкта).
- **Посилання на файл із сезонними погодинними графіками** (Google Sheets, опційно) — додає сторінку «05» з порівнянням погодинної генерації за січень / квітень / липень / жовтень.
- Реквізити КП: об'єкт, адреса, замовник, № КП, термін дії, ставка ПДВ, передоплата, тариф заміщення тощо (мають розумні значення за замовчуванням).

На виході — готове КП, яке друкують у браузері (кнопка **«Друк / зберегти як PDF»**).

## Формати, режими та перемикачі

**Формат КП:**

- **Презентація** — альбомні «слайди» (обкладинка, «Чому ESCORE», технічне рішення, бюджет, гарантія, картка менеджера тощо). Зберігається у PDF через `html2canvas` + `jsPDF` — тобто PDF виглядає точно так само, як прев'ю на екрані, незалежно від того, як конкретний браузер друкує CSS.
- **Документ** — компактний портретний формат: логотип, короткий технічний вступ, далі все таблицями. Друкується **нативним браузерним друком** (`window.print()`), бо контент природно переливається між сторінками.

**Тип розрахунку:**

- **ПДВ** — стандартний розрахунок з ПДВ (вкладка `ПДВ`), у бюджеті показуються три рядки підсумку: разом без ПДВ, ПДВ, загальна вартість з ПДВ.
- **C** — розрахунок без ПДВ для приватних клієнтів. Читає окрему номенклатурну вкладку (її назва — у `js/config.js`, `SHEET_TAB_CASH`), у бюджеті — один згорнутий рядок «Загальна вартість».

**Додаткові перемикачі:**

- **Розширений бюджет** — розшифровує групу «Витратні матеріали» на сторінці бюджету по комплектуючим (Автоматика захисту AC, DC, кабельно-провідникова продукція), дані беруться з вкладки «Кошторис».
- **Без сонячних панелей** — для розрахунків без генерації (напр. лише акумуляторна система): ховає картки та графіки, пов'язані з генерацією, і прибирає сторінку «Фінансові показники».
- **Розділи КП** (наразі діє лише для формату «Документ») — чотири чекбокси: технічне рішення / характеристики, фінансові показники, бюджет реалізації, гарантійний термін. Якщо менеджер сам їх не чіпав, розділ без відповідних даних у файлі знімається автоматично.

Сайт уже розгорнутий: `Settings → Pages`, джерело — гілка `main`, коренева тека. Публічна адреса — https://escoreodessa.github.io/kp-generator/.

Хостинг безкоштовний. Будь-яке оновлення файлів у гілці `main` (через веб-редактор GitHub або `git push`) за кілька хвилин автоматично публікується на живому сайті.

Локально `index.html` можна відкрити прямо в браузері (`file://`), без сервера.

## Налаштування Google API (один раз)

Сайт читає дані напряму з Google через публічний API-ключ (без OAuth/логіну). Використовується той самий ключ і проєкт Google Cloud, що й для Maps у `ses-calculator` — на ньому увімкнені **Google Sheets API** (розрахунок і сезонні графіки) та **Google Drive API** (звіт PVsyst.pdf).

Якщо колись доведеться перевипускати ключ:

1. Google Cloud Console → потрібний проєкт.
2. `APIs & Services → Library` → увімкнути **Google Sheets API** і **Google Drive API**.
3. `APIs & Services → Credentials` → у ключа в списку дозволених API (якщо ключ обмежений) додати обидва — Sheets і Drive. **Важливо:** увімкнути API на рівні проєкту недостатньо — його треба ще додати до списку дозволених саме для цього ключа, інакше запити мовчки повертають 403.
4. Вставити ключ у `js/config.js` (`GOOGLE_API_KEY`).

## Вимоги до файлів-джерел

- **Доступ «для всіх, у кого є посилання» (перегляд).** Сайт не використовує особистий логін Google — лише публічний доступ за посиланням. Без цього API-ключ не прочитає файл.
- **Тільки нативні Google Sheets, не завантажені `.xlsx`.** Якщо файл — це `.xlsx`, який просто відкрито/переглядається в Google Таблицях (біля назви бейдж «XLSX»), Sheets API його **не прочитає** (помилка HTTP 400). Треба `Файл → Зберегти як Google Таблиці`, і вже конвертовану копію відкрити «за посиланням» та дати її посилання генератору. Це стосується і файлу-розрахунку, і файлу з сезонними графіками.

## Якщо КП перестало підтягувати цифри

Парсер у `js/sheets.js` здебільшого шукає дані **за текстом заголовків**, а не за жорсткими адресами комірок — щоб дрібні зміни у файлі (нова колонка, інший порядок) не ламали генератор. Частина показників (фінансовий блок, підсумки бюджету) навмисно читається за **фіксованими адресами комірок** — там, де структура шаблону стабільна.

Якщо якісь цифри зникли або стали нульовими:

- Перевір **назви вкладок** у `js/config.js` (`SHEET_TAB_PDV`, `SHEET_TAB_MODEL`, `SHEET_TAB_CASH`) — вони мають точно збігатися з назвами вкладок у файлі.
- Переконайся, що файл — **нативний Google Sheet**, а не `.xlsx` (див. вимоги вище).
- Для показників із фіксованих комірок перевір, чи не змістилася структура шаблону (напр. вставлений зайвий рядок у категорію бюджету зсуває підсумкові комірки).
- Ключові слова пошуку заголовків — у коментарях у `js/sheets.js`.

## Структура проєкту

| Файл | Призначення |
|---|---|
| `index.html` | Форма (посилання, режими, перемикачі, реквізити) + контейнер документа |
| `style.css` | Брендовані стилі + верстка під друк (A4, розриви сторінок, режими «слайди» і «документ») |
| `js/config.js` | API-ключ, назви вкладок, реквізити компанії, картка менеджера, бізнес-правила за замовчуванням |
| `js/logo.js` | Логотипи ESCORE (base64) |
| `js/sheets.js` | Читання Google Sheets: номенклатура (ПДВ / «C»), фінансова модель, кошторис — через Sheets API |
| `js/drive.js` | Завантаження файлу з Google Drive (звіт PVsyst.pdf) через Drive API v3 |
| `js/pdf-report.js` | Рендер потрібної сторінки PDF-звіту в картинку (pdf.js) з опційним обрізанням |
| `js/seasonal.js` | Читання файлу із сезонними погодинними графіками (окремий Google Sheet) |
| `js/images.js` | Читання завантажених зображень |
| `js/kp-render.js` | Побудова КП: формат «Презентація» (слайди + діаграми Chart.js) і формат «Документ» (таблиці) |
| `js/app.js` | Зшиває форму, режими, парсинг, рендер і збереження у PDF |
| `assets/` | Статичні зображення (фон обкладинки, білий логотип, сертифікат, фото менеджера, лого партнерів) |

Зовнішні бібліотеки (через CDN): Chart.js, pdf.js, html2canvas, jsPDF.
